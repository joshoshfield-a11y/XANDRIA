# CHAPTER 3: THE ALPHA SYNTAX ENGINE

## 3.0 The Master Alpha Syntax

All Alpha Hyper-Prompts follow the unified structure:

```
[CONTEXT: Domain] >> [OPERATOR: Action] << [INPUT: Data] ? [LOGIC: Condition] ! [PRIORITY: Level] @ [CONSTRAINT: Boundaries]
```

### Component Definitions:

| Component | Symbol | Purpose | Example |
|-----------|--------|---------|---------|
| **Context** | `[]` | Defines the operational domain | `[CONTEXT: Systems Architect]` |
| **Operator** | `>>` | The primary action directive | `>> [OPERATOR: Analyze]` |
| **Input** | `<<` | The data or content to process | `<< [INPUT: Code Snippet]` |
| **Logic** | `?` | Conditional branching | `? [LOGIC: If Error Exists]` |
| **Priority** | `!` | Force level and urgency | `! [PRIORITY: CRITICAL]` |
| **Constraint** | `@` | Boundaries and limitations | `@ [CONSTRAINT: 500 tokens max]` |

| **Injection** | `>>` | Alpha Input | Sends data to processing engine |
| **Extraction** | `<<` | Alpha Output | Retrieves processed information |

| **Conditional** | `?` | Alpha Branch | Logic-based path selection |
| **Iteration** | `@` | Alpha Loop | Controlled repetition |

| **Force** | `!` | Alpha Command | Absolute priority override |
